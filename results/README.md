# Results

All experimental results in one place. 26 trained runs, four metrics each.

## Documents in this folder

| File | What's in it |
|---|---|
| **`PARETO.md`** | Top-5 by each of the four probe metrics (alpha lin, alpha kNN, zeta lin, zeta kNN), plus the Pareto-optimal frontier on the joint (alpha kNN, zeta kNN) plane. **Start here if you want the headline numbers.** |
| **`ABLATION_EFFECTS.md`** | Two big tables (one per parameter) covering every paired-comparison ablation we have data for: backbone, target encoder, loss family, VICReg cov term, all three lambda sweeps, var_weight, cov_weight magnitude, routing. Each row is a single design-axis toggle with a verdict label. |
| **`RUN_INVENTORY.md`** | Every run as a row: full sortable table of all 26 cells, then grouped views per routing, then a hyperparameter table. The raw record. |
| **`METRICS.md`** | Per-run metadata from `final.json`: parameter counts, wall time, final training losses. Good for compute accounting and the report's reproducibility checklist. |
| `_runs.json` | Machine-readable raw inventory underlying the above docs. Regenerated by `scripts/_gen_results_docs.py` whenever new runs land. |

## Where the data comes from

- **13 EMA-target runs** under `REFACTORED_CODEBASE/runs/*_2026*/` (cells from the 2026-04-28 / 04-29 ablation queue).
- **13 shared-encoder runs** under `../runs/*` (the original SIGReg + VICReg sweep from 2026-04-21 through 2026-04-24).

Both groups are evaluated with the same pipeline (`scripts/eval.py` -> `src/eval/{linear_probe,knn_regression}.py`) using train-split z-score statistics applied to val/test labels. Constant-mean baseline = 1.0; lower MSE = better.

## Project leaders at a glance

| Metric | Best run | Value |
|---|---|---|
| alpha kNN | `baseline_cnn_ema_vicreg_no_cov` | **0.0131** |
| alpha linear | `baseline_cnn_ema_vicreg_no_cov` | **0.0195** |

CNN dominates alpha; ViT dominates zeta. Same routing (baseline, all-channel-to-all-channel), same target type (EMA), same loss family (VICReg) — only backbone and the covariance term differ.

## Regenerating

These docs are rebuilt from the live run dirs at any time by running:

```
cd REFACTORED_CODEBASE
python scripts/_gen_results_docs.py
```

If you train new runs (in either `REFACTORED_CODEBASE/runs/` or `../runs/`), just rerun the generator and `RUN_INVENTORY.md`, `METRICS.md`, `PARETO.md`, and `_runs.json` will refresh. `ABLATION_EFFECTS.md` was hand-finalized with cross-table commentary; if it needs an update, regenerate the body via the same script then re-port the summary section.
